## AI・生成AIツール

### ChatGPT ⭐⭐⭐
OpenAIが開発した対話型AI。コード生成、デバッグ支援、技術相談などに活用されます。

**WEB開発での活用:**
- コードの自動生成
- エラーの原因特定と修正案提示
- コードレビュー
- ドキュメント作成
- 技術的な質問への回答

**実用例:**
```
プロンプト: 「レスポンシブなナビゲーションバーのHTMLとCSSを書いて」
→ 即座にコードが生成される
```

**関連用語:** [Copilot](#github-copilot-⭐⭐⭐), [プロンプト](#プロンプト-⭐⭐⭐), [Claude](#claude-⭐⭐)

### GitHub Copilot ⭐⭐⭐
GitHubとOpenAIが開発したAIコーディングアシスタント。エディタ内でリアルタイムにコード補完します。

**主な機能:**
- コードの自動補完
- コメントからコード生成
- 複数の提案を表示
- テストコードの生成

**使用例:**
```javascript
// コメントを書くだけでコードが提案される
// 配列から重複を削除する関数
// → Copilotが自動でコード提案
const removeDuplicates = (arr) => [...new Set(arr)];
```

**料金:** 月額$10（学生・OSS開発者は無料）

**関連用語:** [ChatGPT](#chatgpt-⭐⭐⭐), [AIアシスタント](#aiアシスタント-⭐⭐⭐), [VSCode](#vscode-⭐⭐⭐)

### Claude ⭐⭐
Anthropicが開発した対話型AI。長文の理解に優れ、コードレビューやドキュメント作成に適しています。

**特徴:**
- 長い文脈の理解
- 安全性重視の設計
- コードの詳細な説明
- 段階的な問題解決

**WEB開発での活用:**
- 大規模なコードベースの分析
- アーキテクチャ設計の相談
- セキュリティレビュー

**関連用語:** [ChatGPT](#chatgpt-⭐⭐⭐), [AIアシスタント](#aiアシスタント-⭐⭐⭐)

### Midjourney ⭐⭐
AI画像生成ツール。テキストプロンプトから高品質な画像を生成します。

**WEBデザインでの活用:**
- ヒーローイメージの生成
- アイコン・イラスト作成
- モックアップ用の素材
- デザインアイデアの視覚化

**使用例:**
```
プロンプト: "modern minimalist website hero image,
tech startup, blue gradient --ar 16:9"
→ プロフェッショナルなヒーロー画像が生成
```

**関連用語:** [Stable Diffusion](#stable-diffusion-⭐), [DALL-E](#dall-e-⭐), [プロンプト](#プロンプト-⭐⭐⭐)

### Figma AI ⭐⭐
Figmaに統合されたAI機能。デザイン作業を効率化します。

**主な機能:**
- テキストから画像生成
- 自動レイアウト調整
- デザイン要素の自動整列
- カラーパレット提案

**実用例:**
- "modern login form" → UIコンポーネント生成
- 既存デザインのレスポンシブ対応を自動化

**関連用語:** [Figma](#figma-⭐⭐⭐), [デザインツール](#デザインツール-⭐⭐)

### v0.dev ⭐⭐
Vercelが開発したAI UIジェネレーター。テキストからReactコンポーネントを生成します。

**特徴:**
- プロンプトからUI生成
- Tailwind CSSベース
- コピー&ペースト可能なコード
- リアルタイムプレビュー

**使用例:**
```
プロンプト: "pricing table with 3 tiers"
→ 完全に動作するReactコンポーネントが生成
```

**関連用語:** [Vercel](#vercel-⭐⭐), [React](#react-⭐⭐⭐), [Tailwind CSS](#tailwind-css-⭐⭐)

## AI活用開発

### AIアシスタント ⭐⭐⭐
開発をサポートするAIツールの総称。

**主なツール:**
- コード生成: Copilot, Cursor, Tabnine
- デザイン: Figma AI, Uizard, Galileo AI
- テスト: Codium AI
- ドキュメント: ChatGPT, Claude

**活用シーン:**
- 「この関数の説明を書いて」
- 「このコードをリファクタリングして」
- 「バグの原因を見つけて」

**関連用語:** [ChatGPT](#chatgpt-⭐⭐⭐), [Copilot](#github-copilot-⭐⭐⭐)

### プロンプト (Prompt) ⭐⭐⭐
AIへの指示文。効果的なプロンプトで品質が大きく変わります。

**良いプロンプトの例:**
```
悪い例: "ボタン作って"

良い例: "レスポンシブなプライマリーボタンのReactコンポーネントを作成。
- Tailwind CSSを使用
- ホバー/フォーカス状態を含む
- アクセシビリティ対応
- TypeScriptで型定義"
```

**プロンプトエンジニアリングのコツ:**
- 具体的に指示
- 文脈を提供
- 期待する形式を明示
- 制約条件を明確に

**関連用語:** [ChatGPT](#chatgpt-⭐⭐⭐), [生成AI](#生成ai-⭐⭐⭐)

### 生成AI (Generative AI) ⭐⭐⭐
新しいコンテンツを生成できるAI。

**WEB開発での活用:**
- **コード:** GitHub Copilot, ChatGPT
- **画像:** Midjourney, Stable Diffusion
- **テキスト:** ChatGPT, Claude
- **UI:** v0.dev, Galileo AI

**注意点:**
- 生成コードの検証が必要
- ライセンス・権利関係の確認
- セキュリティリスクのチェック

**関連用語:** [ChatGPT](#chatgpt-⭐⭐⭐), [Midjourney](#midjourney-⭐⭐)

### AIペアプログラミング ⭐⭐
AIをペアプログラミングのパートナーとして活用する手法。

**活用方法:**
```javascript
// 1. コメントで意図を書く
// ユーザー情報を取得してダッシュボードに表示する関数

// 2. Copilotが提案
async function fetchUserDashboard(userId) {
  // AIが残りを補完...
}

// 3. レビューして必要に応じて修正
```

**メリット:**
- 開発速度の向上
- ボイラープレート削減
- 学習効率の向上

**関連用語:** [Copilot](#github-copilot-⭐⭐⭐), [ペアプロ](#ペアプロ-⭐⭐)

## 開発スラング・略語

### プルリク / PR (Pull Request) ⭐⭐⭐
コードの変更をレビュー・マージしてもらうためのリクエスト。

**使用例:**
```
「プルリク出しました」= Pull Requestを作成した
「PRレビューお願いします」
「PRマージしました」
```

**一般的な流れ:**
```bash
git checkout -b feature/new-button
# コード変更
git add .
git commit -m "Add new button component"
git push origin feature/new-button
# GitHubでPR作成
```

**関連用語:** [マージ](#マージ-merge-⭐⭐⭐), [レビュー](#コードレビュー-⭐⭐⭐), [GitHub](#github-⭐⭐⭐)

### マージ (Merge) ⭐⭐⭐
ブランチの変更を統合すること。動詞としても使われます。

**使い方:**
```
「マージしていい?」
「main にマージしました」
「マージコンフリクト起きた」= 競合が発生
```

**関連スラング:**
- マジる = マージする
- コンフリ = マージコンフリクト（競合）

**関連用語:** [プルリク](#プルリクpr-pull-request-⭐⭐⭐), [Git](#git-⭐⭐⭐)

### デプロイ (Deploy) ⭐⭐⭐
コードを本番環境に配置すること。動詞として頻繁に使われます。

**使用例:**
```
「今日デプロイします」
「デプロる」= デプロイする
「デプロ済み」= デプロイ完了
「金曜デプロイは避ける」= 週末前のデプロイは危険
```

**関連用語:** [CI/CD](#cicd-⭐⭐⭐), [本番環境](#本番環境-production-⭐⭐⭐)

### リファクタ / リファクタリング ⭐⭐⭐
動作を変えずにコードを整理・改善すること。

**使用例:**
```
「このコードリファクタしといた」
「リファクタ祭り」= 大規模なリファクタリング
「技術的負債返済のためリファクタ中」
```

**目的:**
- 可読性向上
- 保守性向上
- パフォーマンス改善

**関連用語:** [技術的負債](#技術的負債-⭐⭐), [コードレビュー](#コードレビュー-⭐⭐⭐)

### コミット / コミる ⭐⭐⭐
変更を記録すること。

**スラング:**
```
「コミる」= コミットする
「コミっといて」= コミットしておいて
「コミット粒度」= 1コミットの変更量
「原子的コミット」= 小さく意味のあるコミット
```

**良いコミットメッセージ:**
```bash
# 悪い例
git commit -m "fix"

# 良い例
git commit -m "Fix responsive layout issue on mobile devices"
```

**関連用語:** [Git](#git-⭐⭐⭐), [バージョン管理](#バージョン管理-⭐⭐)

### イシュー / Issue ⭐⭐⭐
GitHubなどで管理するタスクやバグ報告。

**使用例:**
```
「イシュー切っといた」= Issueを作成した
「イシュ立てる」= Issueを作成する
「イシュー消化」= Issueを片付ける
```

**Issueの種類:**
- Bug: バグ報告
- Feature: 機能リクエスト
- Enhancement: 改善提案
- Documentation: ドキュメント

**関連用語:** [GitHub](#github-⭐⭐⭐), [タスク](#タスク-⭐⭐⭐)

### レポ / リポジトリ ⭐⭐⭐
コードを保管する場所。

**使用例:**
```
「レポのURL送って」
「このレポforkした」
「プライベートレポ」= 非公開リポジトリ
```

**関連用語:** [GitHub](#github-⭐⭐⭐), [Git](#git-⭐⭐⭐)

### レスポ / レスポンシブ ⭐⭐⭐
画面サイズに応じた表示の対応。

**使用例:**
```
「レスポ対応した？」
「レスポ確認してください」
「レスポが崩れてる」= レスポンシブ表示が正しくない
```

**関連用語:** [レスポンシブデザイン](#レスポンシブデザイン-⭐⭐⭐), [モバイルファースト](#モバイルファースト-⭐⭐⭐)

### モック / モックアップ ⭐⭐⭐
デザインの試作品。

**使用例:**
```
「モック作った」= デザイン案を作成
「モック見せて」= デザイン案を確認
「ローファイモック」= 簡易的な試作
「ハイファイモック」= 詳細な試作
```

**関連用語:** [プロトタイプ](#プロトタイププロト-⭐⭐), [ワイヤーフレーム](#ワイヤーフレーム-⭐⭐)

### プロトタイプ / プロト ⭐⭐
動作する試作品。

**使用例:**
```
「プロト作って検証しよう」
「プロトで確認」= 試作版で動作確認
```

**関連用語:** [モック](#モックモックアップ-⭐⭐⭐), [MVP](#mvp-⭐⭐)

### WIP (Work In Progress) ⭐⭐⭐
作業中を意味する略語。

**使用例:**
```
「[WIP] ヘッダーコンポーネント作成中」
「WIPのPR出しました」= レビューは後で
「WIP外した」= 作業完了してレビュー可能に
```

**関連用語:** [プルリク](#プルリクpr-pull-request-⭐⭐⭐), [Draft](#draft-⭐⭐)

### LGTM (Looks Good To Me) ⭐⭐⭐
「良さそうです」という承認の意味。

**使用例:**
```
「LGTM!」= 承認します
「2 LGTM もらった」= 2人から承認された
「LGTM後にマージします」
```

**関連用語:** [コードレビュー](#コードレビュー-⭐⭐⭐), [プルリク](#プルリクpr-pull-request-⭐⭐⭐)

### IMO / IMHO ⭐⭐
個人的な意見を述べる時の前置き。

**意味:**
- IMO: In My Opinion（私の意見では）
- IMHO: In My Humble Opinion（私の控えめな意見では）

**使用例:**
```
「IMO、このアプローチの方が良いと思います」
「IMHO、リファクタが必要です」
```

**関連用語:** [FYI](#fyi-⭐⭐⭐), [TBD](#tbd-⭐⭐)

### FYI (For Your Information) ⭐⭐⭐
「参考までに」という意味。

**使用例:**
```
「FYI: 本番環境にデプロイしました」
「FYI、この記事参考になるかも」
```

**関連用語:** [IMO](#imoimho-⭐⭐)

### TBD (To Be Determined) ⭐⭐
「後で決める」「未定」という意味。

**使用例:**
```
「デザイン: TBD」
「リリース日: TBD」
「TBDの部分を埋めていく」
```

**関連用語:** [TODO](#todo-⭐⭐⭐), [WIP](#wip-work-in-progress-⭐⭐⭐)

### TODO / ToDo ⭐⭐⭐
「やること」を意味する。コード内のコメントとしても使用。

**使用例:**
```javascript
// TODO: エラーハンドリング追加
// TODO: パフォーマンス最適化
// FIXME: このロジックは修正が必要
// HACK: 暫定的な実装
```

**関連用語:** [タスク](#タスク-⭐⭐⭐), [イシュー](#イシューissue-⭐⭐⭐)

### デグレ / デグレード ⭐⭐
以前動いていた機能が動かなくなること。

**使用例:**
```
「デグレしてる！」= 機能が壊れた
「デグレないか確認」= 既存機能のテスト
「デグレ防止のため自動テスト追加」
```

**関連用語:** [バグ](#バグ-bug-⭐⭐⭐), [リグレッションテスト](#リグレッションテスト-⭐)

### バグる ⭐⭐⭐
バグが発生すること。

**使用例:**
```
「バグってる」= 正常に動作していない
「バグった」= バグが発生した
「IEでバグる」= IEで不具合が出る
```

**関連用語:** [バグ](#バグ-bug-⭐⭐⭐), [デバッグ](#デバッグ-⭐⭐⭐)

### 落ちる ⭐⭐⭐
システムやアプリが停止すること。

**使用例:**
```
「サーバーが落ちた」= サーバーダウン
「ビルドが落ちる」= ビルド失敗
「テストが落ちる」= テスト失敗
```

**関連用語:** [ダウン](#ダウン-⭐⭐), [クラッシュ](#クラッシュ-⭐⭐)

### 重い ⭐⭐⭐
動作が遅いこと。

**使用例:**
```
「この画面重い」= 表示が遅い
「重い処理」= 時間がかかる処理
「重くならないように最適化」
```

**関連用語:** [パフォーマンス](#パフォーマンス-⭐⭐⭐), [最適化](#最適化-⭐⭐)

## チーム開発用語

### コードレビュー ⭐⭐⭐
他の開発者がコードを確認すること。

**使用例:**
```
「コードレビューお願いします」
「レビュー修正しました」
「セルフレビュー」= 自分でコードを見直す
```

**レビューのポイント:**
- ロジックの正確性
- 可読性
- パフォーマンス
- セキュリティ
- テストの充実度

**関連用語:** [プルリク](#プルリクpr-pull-request-⭐⭐⭐), [LGTM](#lgtm-looks-good-to-me-⭐⭐⭐)

### ペアプロ / ペアプログラミング ⭐⭐
2人で1つのコードを書く開発手法。

**役割:**
- **ドライバー**: コードを書く人
- **ナビゲーター**: 方向性を示す人

**メリット:**
- 知識共有
- コード品質向上
- 問題解決の高速化
- オンボーディング効果

**使用例:**
```
「ペアプロしませんか？」
「今日はペアプロデー」
```

**関連用語:** [モブプロ](#モブプロ-⭐), [レビュー](#コードレビュー-⭐⭐⭐)

### スタンドアップ / 朝会 ⭐⭐
短時間の進捗共有ミーティング。

**話す内容（3つの質問）:**
1. 昨日やったこと
2. 今日やること
3. 困っていること・ブロッカー

**使用例:**
```
「朝会で共有します」
「スタンドアップの時間です」
「デイリースタンドアップ」= 毎日の朝会
```

**関連用語:** [スクラム](#スクラム-⭐⭐), [アジャイル](#アジャイル-⭐⭐)

### スプリント ⭐⭐
1〜4週間の開発サイクル。アジャイル開発で使われます。

**使用例:**
```
「今スプリントの目標」
「スプリント計画」= スプリントの最初の計画会議
「スプリントレビュー」= スプリント終了時の振り返り
```

**関連用語:** [アジャイル](#アジャイル-⭐⭐), [スクラム](#スクラム-⭐⭐)

### アジャイル (Agile) ⭐⭐
柔軟で反復的なソフトウェア開発手法。

**特徴:**
- 短いサイクルでのリリース
- 変更への柔軟な対応
- 継続的な改善
- チームでのコミュニケーション重視

**対比:**
- **アジャイル**: 柔軟、反復的
- **ウォーターフォール**: 計画重視、段階的

**関連用語:** [スクラム](#スクラム-⭐⭐), [スプリント](#スプリント-⭐⭐)

### スクラム (Scrum) ⭐⭐
アジャイル開発のフレームワークの一つ。

**役割:**
- **プロダクトオーナー**: 要件を決める
- **スクラムマスター**: チームを支援
- **開発チーム**: 実装を担当

**イベント:**
- スプリント計画
- デイリースタンドアップ
- スプリントレビュー
- レトロスペクティブ（振り返り）

**関連用語:** [アジャイル](#アジャイル-agile-⭐⭐), [スプリント](#スプリント-⭐⭐)

### オンボーディング ⭐⭐
新メンバーがチームに馴染むためのプロセス。

**内容:**
- 開発環境のセットアップ
- コードベースの説明
- チームの働き方の共有
- 最初のタスクのアサイン

**使用例:**
```
「新メンバーのオンボーディング資料」
「オンボーディング期間」
```

**関連用語:** [ドキュメント](#ドキュメント-⭐⭐⭐), [ナレッジ共有](#ナレッジ共有-⭐⭐)

### タスク ⭐⭐⭐
作業の単位。

**使用例:**
```
「タスク切り出し」= 大きな作業を小さく分割
「タスク消化」= タスクを完了させる
「タスクの粒度」= タスクの大きさ
```

**管理ツール:**
- Jira
- Trello
- GitHub Issues
- Asana

**関連用語:** [イシュー](#イシューissue-⭐⭐⭐), [チケット](#チケット-⭐⭐)

### ブロッカー ⭐⭐
作業を進める上での障害。

**使用例:**
```
「ブロッカーがあります」= 進められない理由がある
「ブロッカー解消」= 障害を取り除く
「ブロッカーは特にないです」
```

**よくあるブロッカー:**
- 仕様が不明確
- 他のタスクの完了待ち
- 技術的な問題
- リソース不足

**関連用語:** [タスク](#タスク-⭐⭐⭐), [デペンデンシー](#デペンデンシー-⭐)

## モダンWEB技術

### SPA (Single Page Application) ⭐⭐⭐
1つのHTMLページで動作するWEBアプリ。

**特徴:**
- ページ遷移が高速
- ネイティブアプリのようなUX
- JavaScriptで動的に更新

**主なフレームワーク:**
- React
- Vue.js
- Angular

**例:**
```
Gmail、Twitter、Figmaなどが代表例
```

**関連用語:** [React](#react-⭐⭐⭐), [CSR](#csr-client-side-rendering-⭐⭐)

### SSR (Server Side Rendering) ⭐⭐
サーバー側でHTMLを生成して返す手法。

**メリット:**
- 初期表示が速い
- SEOに有利
- SNSシェアで適切なOGP表示

**使用するフレームワーク:**
- Next.js（React）
- Nuxt.js（Vue）
- SvelteKit（Svelte）

**関連用語:** [CSR](#csr-client-side-rendering-⭐⭐), [SSG](#ssg-static-site-generation-⭐⭐)

### SSG (Static Site Generation) ⭐⭐
ビルド時に静的HTMLを生成する手法。

**メリット:**
- 表示が超高速
- サーバー負荷が低い
- セキュリティが高い
- ホスティングコストが安い

**使用例:**
- ブログ
- ドキュメントサイト
- ランディングページ

**ツール:**
- Next.js
- Gatsby
- Astro
- Hugo

**関連用語:** [SSR](#ssr-server-side-rendering-⭐⭐), [Jamstack](#jamstack-⭐⭐)

### CSR (Client Side Rendering) ⭐⭐
ブラウザ側（クライアント）でHTMLを生成する手法。

**特徴:**
- JavaScriptで動的に描画
- インタラクティブ
- 初期表示は遅め

**関連用語:** [SPA](#spa-single-page-application-⭐⭐⭐), [SSR](#ssr-server-side-rendering-⭐⭐)

### API ⭐⭐⭐
アプリケーション間でデータをやり取りする仕組み。

**使用例:**
```javascript
// REST API でデータ取得
fetch('https://api.example.com/users')
  .then(res => res.json())
  .then(data => console.log(data));
```

**種類:**
- **REST API**: 最も一般的
- **GraphQL**: 柔軟なデータ取得
- **WebSocket**: リアルタイム通信

**関連用語:** [REST](#rest-⭐⭐⭐), [GraphQL](#graphql-⭐⭐), [エンドポイント](#エンドポイント-⭐⭐⭐)

### REST API ⭐⭐⭐
HTTPを使ったAPI設計スタイル。

**HTTPメソッド:**
```
GET    /users      # ユーザー一覧取得
GET    /users/123  # 特定ユーザー取得
POST   /users      # ユーザー作成
PUT    /users/123  # ユーザー更新
DELETE /users/123  # ユーザー削除
```

**関連用語:** [API](#api-⭐⭐⭐), [エンドポイント](#エンドポイント-⭐⭐⭐)

### GraphQL ⭐⭐
Facebookが開発したAPIのクエリ言語。

**特徴:**
```graphql
# 必要なデータだけ取得
query {
  user(id: "123") {
    name
    email
    posts {
      title
    }
  }
}
```

**メリット:**
- 過不足ないデータ取得
- 1回のリクエストで複数リソース取得
- 型安全

**関連用語:** [API](#api-⭐⭐⭐), [REST](#rest-⭐⭐⭐)

### PWA (Progressive Web App) ⭐⭐
ネイティブアプリのような体験を提供するWEBアプリ。

**特徴:**
- オフライン動作
- ホーム画面に追加可能
- プッシュ通知
- 高速な読み込み

**必要な要素:**
- Service Worker
- Manifest ファイル
- HTTPS

**関連用語:** [Service Worker](#service-worker-⭐), [Manifest](#manifest-⭐)

### Jamstack ⭐⭐
JavaScript、API、Markupで構成される開発アーキテクチャ。

**特徴:**
- 静的サイトをベースに
- APIで動的機能を追加
- Git ベースのワークフロー
- CDN配信

**使用ツール:**
- Netlify
- Vercel
- Cloudflare Pages

**関連用語:** [SSG](#ssg-static-site-generation-⭐⭐), [CDN](#cdn-⭐⭐)

## 開発環境・ツール

### CI/CD ⭐⭐⭐
継続的インテグレーション・継続的デリバリー。

**CI (Continuous Integration):**
- コードを頻繁にマージ
- 自動テスト実行
- ビルドの自動化

**CD (Continuous Delivery/Deployment):**
- 自動デプロイ
- 本番環境への継続的リリース

**ツール:**
- GitHub Actions
- GitLab CI/CD
- CircleCI
- Jenkins

**関連用語:** [デプロイ](#デプロイ-deploy-⭐⭐⭐), [自動化](#自動化-⭐⭐)

### Docker ⭐⭐⭐
アプリケーションをコンテナ化するプラットフォーム。

**メリット:**
- 環境の統一
- 「自分の環境では動く」問題の解消
- 簡単なデプロイ

**基本コマンド:**
```bash
docker build -t myapp .
docker run -p 3000:3000 myapp
docker-compose up
```

**関連用語:** [コンテナ](#コンテナ-⭐⭐), [DevOps](#devops-⭐⭐)

### コンテナ (Container) ⭐⭐
アプリケーションと依存関係をパッケージ化したもの。

**特徴:**
- 軽量
- 起動が速い
- 環境の再現性

**関連用語:** [Docker](#docker-⭐⭐⭐), [Kubernetes](#kubernetes-⭐)

### npm / yarn / pnpm ⭐⭐⭐
JavaScriptのパッケージマネージャー。

**使用例:**
```bash
# パッケージインストール
npm install react
yarn add react
pnpm add react

# 開発サーバー起動
npm run dev
yarn dev
pnpm dev
```

**関連用語:** [パッケージ](#パッケージ-⭐⭐⭐), [Node.js](#nodejs-⭐⭐⭐)

### Webpack / Vite / Turbopack ⭐⭐
モジュールバンドラー。複数ファイルを1つにまとめます。

**比較:**
- **Webpack**: 定番、設定が複雑
- **Vite**: 高速、モダン
- **Turbopack**: Next.js 13+、Rust製で超高速

**関連用語:** [ビルド](#ビルド-⭐⭐⭐), [バンドル](#バンドル-⭐⭐)

### Vercel ⭐⭐
Next.js を開発している会社のホスティングサービス。

**特徴:**
- Git連携で自動デプロイ
- プレビュー環境自動生成
- エッジでの高速配信
- 無料プランあり

**使用例:**
```bash
# デプロイ
vercel
```

**関連用語:** [デプロイ](#デプロイ-deploy-⭐⭐⭐), [Netlify](#netlify-⭐⭐)

### Figma ⭐⭐⭐
ブラウザベースのデザインツール。

**特徴:**
- リアルタイムコラボレーション
- プロトタイプ作成
- デベロッパーハンドオフ
- プラグイン豊富

**エンジニアの活用:**
- デザインの確認
- CSSコードの取得
- 画像のエクスポート

**関連用語:** [デザインツール](#デザインツール-⭐⭐), [プロトタイプ](#プロトタイププロト-⭐⭐)

### TypeScript ⭐⭐⭐
Microsoftが開発したJavaScriptに型を追加した言語。

**メリット:**
```typescript
// 型安全
interface User {
  id: number;
  name: string;
  email: string;
}

function getUser(id: number): User {
  // エディタで補完が効く
  // 型エラーをコンパイル時に検出
}
```

**関連用語:** [JavaScript](#javascript-⭐⭐⭐), [型安全](#型安全-⭐⭐)

### React ⭐⭐⭐
Metaが開発したUIライブラリ。

**特徴:**
- コンポーネントベース
- 仮想DOM
- 豊富なエコシステム
- 大規模開発に適している

**基本例:**
```jsx
function Button({ text, onClick }) {
  return (
    <button onClick={onClick}>
      {text}
    </button>
  );
}
```

**関連用語:** [Vue.js](#vuejs-⭐⭐), [コンポーネント](#コンポーネント-⭐⭐⭐)

### Tailwind CSS ⭐⭐⭐
ユーティリティファーストのCSSフレームワーク。

**特徴:**
```html
<div class="flex items-center justify-center h-screen bg-gray-100">
  <button class="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600">
    ボタン
  </button>
</div>
```

**メリット:**
- 高速な開発
- CSS ファイルが不要
- 一貫性のあるデザイン

**関連用語:** [CSS](#css-⭐⭐⭐), [ユーティリティファースト](#ユーティリティファースト-⭐)
